## 3. 本轮整改过的阻塞项

### 阻塞项 1：抓取器候选池策略

原问题：

1. 按 RSS 源顺序抓取
2. 一边抓一边凑满 `10` 条就停止
3. 后面的 RSS 源可能还没参与候选竞争

整改后：

1. 先完成多源抓取
2. 再去重
3. 再按 `published_at` 倒序取最新候选

### 阻塞项 2：WeCom 超长消息处理

原问题：

1. 按字符数截断
2. 中文多字节场景下仍可能超限

整改后：

1. 按 UTF-8 字节安全截断
2. 避免因为单条消息超长直接失败
